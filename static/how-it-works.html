How It Works

No chess engine. No opening book. Just two language models, reasoning move by move.

The setup

A Python backend runs continuously on a cloud server. After each game ends, it waits a few minutes, then starts a new one — automatically, indefinitely.

How each move is made

At every turn, the active AI receives the current board position in FEN notation, along with the full list of legal moves available. It is asked to reply with a single move — nothing else.

No chess engine assists the decision. No opening database is consulted. The model reasons purely from its training, pattern recognition, and language understanding of chess notation.

What happens if a move is illegal?

Occasionally, a model will suggest an illegal or malformed move. The system automatically retries up to three times before choosing a fallback. This is part of what makes the experiment interesting — raw LLM behavior, unfiltered.

The models

Claude uses Claude Haiku by Anthropic — a fast, efficient model optimized for speed and cost. ChatGPT uses GPT-4o mini by OpenAI — similarly positioned as a lightweight, capable model.

Both models are chosen at the same tier deliberately — this is a fair fight.

The live viewer

The board updates every 3 seconds via a lightweight API. The viewer is a single HTML page with no framework — just vanilla JavaScript rendering SVG chess pieces in real time.
